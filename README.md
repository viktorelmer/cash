# Cash

Спокойный офлайн-first трекер личных финансов (PWA). Учёт расходов и доходов, бюджеты, цели, подписки, аналитика — без регистрации и без своего бэкенда.

**Языки интерфейса:** English, Русский, Беларуская.

**Демо:** [viktorelmer.github.io/cash](https://viktorelmer.github.io/cash/)

---

## Возможности

- **Расходы** — быстрое добавление, категории, теги, пресеты
- **Доходы** — зарплата, налоги, планы выплат с pro-rata по рабочим дням
- **Подписки** — учёт периодических платежей и отметка «оплачено»
- **Цели** — накопления с прогрессом и прогнозом
- **Бюджет** — лимиты на месяц (общий и по категориям)
- **Аналитика** — графики, топ категорий, недельные и месячные сводки
- **Мультивалютность** — основная валюта в настройках, пересчёт по курсу
- **Резервная копия** — экспорт/импорт JSON на устройстве

## Куда ходит приложение (сеть)

У Cash **нет своего сервера**. Данные пользователя никуда не отправляются. Единственные исходящие запросы — публичные ресурсы в браузере:

| Источник | Зачем | Когда |
|----------|--------|--------|
| [Frankfurter API](https://frankfurter.dev/) (`api.frankfurter.dev`) | Курсы валют (v2 `/rates`) | При старте, если курсы устарели; при смене основной валюты; по кнопке «Обновить курсы» в настройках |
| [Google Fonts](https://fonts.google.com/) (`fonts.googleapis.com`, `fonts.gstatic.com`) | Шрифт Inter | При загрузке страницы (кэшируется service worker'ом) |

**Не передаётся:** суммы, категории, заметки, цели, подписки, настройки, идентификаторы пользователя — **ничего из ваших финансов**.

Запрос к Frankfurter содержит только код базовой валюты и список нужных валют, без персональных данных.

## Хранение данных

Все данные хранятся **строго локально** в браузере на вашем устройстве:

| Хранилище | Содержимое |
|-----------|------------|
| **IndexedDB** (`cash-db`, через [Dexie](https://dexie.org/)) | Расходы, доходы, подписки, цели, категории, бюджеты, пресеты, планы зарплаты, настройки |
| **Cache Storage** (service worker) | Статические файлы приложения (JS, CSS, иконки) для офлайн-работы PWA |

### Таблицы базы

- `expenses`, `incomes`, `subscriptions`, `goals`, `categories`, `budgets`, `presets`, `salaryPlans`, `settings`

### Резервное копирование

В **Настройки → Резервная копия** можно экспортировать все данные в JSON-файл и восстановить их на другом устройстве. Файл остаётся у вас — приложение его никуда не загружает.

### Удаление данных

- Сброс в настройках («Опасная зона») очищает локальную базу
- Удаление данных сайта в браузере удаляет IndexedDB и кэш PWA

### GitHub Pages

Деплой: push в `main` → GitHub Actions собирает `dist` и публикует его.

1. **Settings → Pages → Build and deployment**
2. Выберите один из вариантов:
   - **Source: GitHub Actions** (рекомендуется), или
   - **Source: Deploy from branch → `gh-pages` → `/ (root)`**
3. URL: `https://<username>.github.io/cash/`

> Если видите пустую страницу — скорее всего Pages отдаёт исходники из `main`, а не сборку. Переключите источник на **GitHub Actions** или ветку **`gh-pages`**.

Локальная проверка:

```bash
GITHUB_PAGES=true npm run build
npx serve dist
```

Опционально: URL репозитория для ссылки в приложении задаётся через `VITE_REPOSITORY_URL` в `.env`.

## Структура (кратко)

```
src/
  app/           — роутинг, корневой App
  components/    — UI и общие компоненты
  features/      — доменная логика (расходы, цели, подписки…)
  hooks/         — React-хуки
  i18n/          — локализация (en, ru, be)
  lib/           — БД, курсы валют, форматирование
  pages/         — экраны приложения
  stores/        — глобальное состояние (настройки)
```
